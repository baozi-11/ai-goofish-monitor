# 闲鱼账号限制排查命令记录

本文记录 Linux / Docker 环境下验证闲鱼账号、登录态、IP、无头浏览器指纹是否被限制的常用命令。

## 1. Docker 环境确认

本项目的 `docker-compose.yaml` 服务名是 `app`，容器名默认是 `ai-goofish-monitor-app`。

查看容器状态：

```bash
docker compose ps
```

进入容器检查 Python：

```bash
docker compose exec app python --version
docker compose exec app python3 --version
```

如果宿主机提示 `Command 'python' not found`，说明你是在宿主机执行了 Python 命令。Docker 部署时应通过 `docker compose exec` 进入容器执行。

如果宿主机提示 `no configuration file provided: not found`，说明当前目录没有 `docker-compose.yaml`。先进入项目目录，或用 `-f` 指定 compose 文件：

```bash
cd /你的项目路径/ai-goofish-monitor
docker compose ps
```

```bash
docker compose -f /你的项目路径/ai-goofish-monitor/docker-compose.yaml ps
```

## 2. Docker 内执行监控任务

在宿主机执行下面命令，让任务在容器内运行：

```bash
docker compose exec -T app python spider_v2.py --task-name "dim十字绣" --debug-limit 1
```

如果容器内也没有 `python` 命令，改用：

```bash
docker compose exec -T app python3 spider_v2.py --task-name "dim十字绣" --debug-limit 1
```

## 3. 可见模式验证

适用于有桌面环境的 Linux 机器。可见模式能直接观察页面是否跳登录、出验证、空白或正常展示商品。

宿主机直接运行：

```bash
RUN_HEADLESS=false LOGIN_IS_EDGE=false python3 spider_v2.py --task-name "dim十字绣" --debug-limit 1
```

Docker 容器内运行：

```bash
docker compose exec -T -e RUN_HEADLESS=false -e LOGIN_IS_EDGE=false app python spider_v2.py --task-name "dim十字绣" --debug-limit 1
```

判断结果：

- 跳转登录、扫码、重新验证：登录态文件大概率失效。
- 出现滑块、安全验证、验证弹窗：账号、IP 或浏览器环境被风控。
- 页面能正常展示商品，但脚本仍然提示搜索响应超时：代码监听的搜索接口规则可能过期。
- 页面空白、一直加载或没有商品接口：优先排查 IP、账号风控和网络连通性。

注意：普通服务器 Docker 通常没有图形显示，可见模式可能无法打开浏览器窗口。无桌面服务器优先使用下一节截图验证。

## 4. 无桌面服务器截图验证

适用于 Linux Docker 服务器。命令会打印闲鱼相关接口、最终页面 URL 和标题，并保存页面截图到宿主机挂载的 `logs/` 目录。

```bash
docker compose exec -T app python - <<'PY'
import asyncio
import json
from playwright.async_api import async_playwright

STATE = "state/baozi-175.json"
KEYWORD = "dim十字绣"
SCREENSHOT = "/app/logs/goofish_account_check.png"

async def main():
    async with async_playwright() as p:
        with open(STATE, "r", encoding="utf-8") as f:
            data = json.load(f)

        storage_state = data
        if isinstance(data, dict) and any(k in data for k in ("env", "headers", "page", "storage")):
            storage_state = {"cookies": data.get("cookies", [])}

        browser = await p.chromium.launch(headless=True, channel="chromium")
        context = await browser.new_context(storage_state=storage_state)
        page = await context.new_page()

        def log_response(response):
            if "h5api.m.goofish.com" in response.url or "passport.goofish.com" in response.url:
                print("API:", response.status, response.request.method, response.url)

        page.on("response", log_response)

        await page.goto("https://www.goofish.com/", wait_until="domcontentloaded", timeout=30000)
        await page.goto(f"https://www.goofish.com/search?q={KEYWORD}", wait_until="domcontentloaded", timeout=60000)
        await page.wait_for_timeout(15000)

        print("FINAL_URL:", page.url)
        print("TITLE:", await page.title())
        await page.screenshot(path=SCREENSHOT, full_page=True)
        print("SCREENSHOT:", SCREENSHOT)
        await browser.close()

asyncio.run(main())
PY
```

## 5. 接口结果判断

重点看命令输出中的 `API:` 行和截图内容：

- 出现 `passport.goofish.com`：登录态失效或账号需要重新登录。
- 截图显示滑块、安全验证、访问受限：账号、IP 或浏览器指纹触发风控。
- 没有 `mtop.taobao.idlemtopsearch.pc.search`：搜索接口未正常触发，可能被拦截或闲鱼接口已变更。
- 出现搜索接口且状态码为 `200`，但正式任务仍超时：优先排查代码里的响应匹配规则和监听时机。

## 6. 对照实验命令

换账号验证：

```bash
cp state/另一个账号.json state/check-account.json
```

然后把脚本中的：

```python
STATE = "state/baozi-175.json"
```

临时改为：

```python
STATE = "state/check-account.json"
```

换 IP 验证：

```bash
curl -4 ip.sb
```

切换到手机热点、代理或另一台服务器后再次执行：

```bash
curl -4 ip.sb
docker compose exec -T -e RUN_HEADLESS=true -e LOGIN_IS_EDGE=false app python spider_v2.py --task-name "dim十字绣" --debug-limit 1
```

## 7. 快速结论表

| 现象 | 优先判断 |
| --- | --- |
| 同 IP 换账号能跑通 | 原账号或原登录态异常 |
| 同账号换 IP 能跑通 | 原 IP 被限制 |
| 可见模式能跑通，无头模式失败 | 无头浏览器指纹被识别 |
| 手动浏览器搜索也异常 | 账号本身被限制或闲鱼侧风控 |
| 搜索接口 200 但脚本超时 | 程序响应匹配条件或监听时机问题 |
